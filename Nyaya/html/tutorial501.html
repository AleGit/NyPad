Boolean Functions

In general a function is a mapping from one set of objects to another set of objects. The natural sum function on n arguments is a mapping from ℕn (set of n-tuples) to ℕ (set of natural numbers). The sum function with three arguments maps (4,8,100) to 112 and we write sum(4,8,100) = 112.

A Boolean function on n arguments is a mappging from {0,1}n to {0,1}.

For example g(0,1) = 1 is a Boolean function which maps the two arguments 0 and 1 to the result 1.

There are two different Boolean functions c() with zero arguments – the constant function c1()=0 and the constant function c2()=1. Often, constant functions are abbreviated to 0 and 1. You can construct four different Boolean functions f(a) with one argument and 16 differnt Boolean functions g(a,b) with 2 arguments. Eight of the functions with two arguments map the 0 and 1 to the result 1.

In general we can construct 22n different Boolean functions with n arguments.

Representations of Boolean functions

Propositional formulas and truth tables are different representations of Boolean functions. When “0” is mapped to “F” (false) and “1” is mapped to “T” (true), the complement finds it‘s counterpart in the logical negation, the product in the logical conjunction, and the sum in the logical disjunction. So every implication free propositional formula can be rewritten as Boolean expression without effort.

Some Boolean functions are highlighted – they have names and a special notation. The complement is a Boolean function with one argument and maps the argument to to it's complement – «0» to «1» and «1» to «0». Usually the complement of «x» is notated as «x». This is impractical for text input and «!x» or «~x» are often used instead. The product «•» maps two arguments to «1», if both arguments are equal to «1». Otherwise (if one or two arguments are «0») the product «•» maps it's argument's to «0». The sum «+» maps two arguments to «0», if both arguments are equal to «0». Otherwise (if one or two arguments are «1») the sum «+» maps it's argument's to «1». The exclusive or «⊕» maps two arguments to «1», if the arguments are different. Otherwise (if the arguments are equal) the exclusive or «⊕» maps it's argument's to «0».

Does a•(b+c) = a•b+a•c look right, but a+b•c = (a+b)•(a+c) does not? Indeed, the second equivalence does not hold for arithmetic expressions, but it does for Boolean expressions, which is easily shown with a truth table.

Precedence and calculation rules

The complement has the highest precedence. «ā•b», «ā+b» and «ā⊕b» are to be interpreted as «(ā)•b», «(ā)+b» and «(ā)⊕b». The product has a higher precedence than the sum and the exclusive or. «a•b+c» means «(a•b)+c». «a•b⊕c means «(a•b)⊕c». The sum has a higher precedence than the exclusive or. «a+b⊕c» is to be read as «(a+b)⊕c».

a•b = b•a    a+b = b+a    a⊕b = b⊕a

ā = a    a+b = a.b    a•b = a+b

a⊕b = a•b + a•b = (a+b) • (a+b)

0⊕a = a = a⊕0    1⊕a = a = a⊕1

a•(b+c) = a•b+a•c    a+b•c = (a+b)•(a+c)
