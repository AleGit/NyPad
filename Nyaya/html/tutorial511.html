Representations of Boolean functions

Propositional formulas and truth tables are different representations of Boolean functions. When “0” is mapped to “F” (false) and “1” is mapped to “T” (true), the complement finds it‘s counterpart in the logical negation, the product in the logical conjunction, and the sum in the logical disjunction. So every implication free propositional formula can be rewritten as Boolean expression without effort. A Boolean expression without xor can be rewritten as propositional formula easily.

Propositional formulas, CNFs, DNFs, truth tables and Boolean expressions are often very complex or representations of Boolean functions. Often validity or satisfiability are not easy to decide in these for propositional formulas and Boolean expressions. Truth tables are expensive to compute and take much storage.

Binary Decision Trees

An efficient representation/definition of Boolean functions can be done with reduced ordered binary decision diagrams. We start with an arbitrary binary decision tree to define an binary function g(x,y,z) with three arguments.

To calculate the value of g(1,1,0) we start at the root node “x on the top”. Because x=1 we follow the solid line to “z“. Because z=0 we follow the dotted line to “y”. Because y=1 we follow the solid line to the result “0”.

x 1111 0000
y 1100 1100
z 1010 1010
g(x,y,z) 1011 1000

We make the following observations

  • The leftmost decision node “x” seams unnessessary, because we can only reach this node if x=0. So we must go to the leftmost “0” and can't go to the leftmost “1”. If we remove the leftmost 1 and replace leftmost “x” with “0“, the same Boolean function is defined.
  • The function is satisfiable. (There is at least one real path to 1)
  • The function is not valid. (There is at least one real path to 0)
  • If x=0 the the result does not depende on z.
  • If x=1 and z=1 the result does not depend on y.
